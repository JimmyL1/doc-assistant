# 台灣科技集團股份有限公司 — 資安事件應變手冊（Incident Response Runbook）

> 文件編號：ISMS-OPS-003 | 版本：v2.1 | 機密等級：C2（內部）
> 維護單位：資訊安全部 | 最後更新：2025-03-15

---

## 1. 目的與適用範圍

本手冊定義 台灣科技集團股份有限公司 在發生資訊安全事件時的標準應變程序，確保所有相關人員能夠迅速、一致地做出回應，將損害降至最低並符合法規要求。

適用範圍：全體員工、協力廠商、委外服務人員。

---

## 2. 事件等級定義

### P1 — 嚴重（Critical）

**觸發條件：**
- 核心服務（API Gateway、資料庫）完全中斷
- 確認的資料外洩（超過1,000筆個資或任何C4機密）
- 勒索軟體感染生產環境
- 內部系統遭外部攻擊者控制

**回應時限：** 15分鐘內通報、1小時內啟動事件應變小組（IRT）

**通報對象：** 資安長（CISO）、技術長（CTO）、執行長、法務

### P2 — 高（High）

**觸發條件：**
- 核心服務效能嚴重降級（>50%）
- 疑似資料外洩（範圍未確定）
- 多台設備受惡意軟體感染
- 特權帳號（Admin/Root）疑似被盜

**回應時限：** 30分鐘內通報、2小時內完成初步評估

**通報對象：** 資安主管、相關系統負責人

### P3 — 中（Medium）

**觸發條件：**
- 個別員工帳號被盜
- 釣魚郵件攻擊（已阻擋，但有員工點擊）
- 非核心系統遭未授權存取
- 設備遺失（含C2以上資料）

**回應時限：** 2小時內通報

**通報對象：** 資安部門

### P4 — 低（Low）

**觸發條件：**
- 釣魚郵件（員工未點擊，主動通報）
- 非敏感設備遺失
- 掃描或偵察行為（未成功入侵）

**回應時限：** 24小時內通報

**通報對象：** 資安部門（一般工單）

## 3. 通報流程

### 3.1 通報管道

| 管道 | 適用情境 |
|------|---------|
| Slack `#security-incident` | P3/P4 事件，工作時間 |
| 電子郵件 security@twtech.com.tw | P3/P4 事件，非緊急 |
| 資安緊急熱線 0800-123-456 | P1/P2 事件，24小時 |
| 直接致電 CISO 張信義（0912-345-678） | P1 事件，無法觸及其他管道時 |

### 3.2 通報內容（SBAR格式）

通報時請提供以下資訊：

- **Situation（狀況）：** 發生了什麼？何時發現？
- **Background（背景）：** 受影響的系統/服務/資料範圍
- **Assessment（評估）：** 初步判斷等級與影響程度
- **Recommendation（建議）：** 已採取的緊急措施

## 4. 處置步驟

### 4.1 P1/P2 事件標準處置流程

**Phase 1：偵測與分類（0-15分鐘）**
1. 確認事件屬實（排除誤報）
2. 初步評估影響範圍
3. 指定事件負責人（Incident Commander）
4. 建立事件專屬 Slack 頻道（`#inc-YYYYMMDD-描述`）

**Phase 2：遏制（15分鐘-2小時）**
1. 隔離受感染系統（網路隔離、暫停服務）
2. 保全證據（記憶體轉儲、日誌備份，禁止關機）
3. 封鎖攻擊來源 IP / 帳號
4. 評估是否需要向外部機構通報（TWCERT、主管機關）

**Phase 3：根除（2-24小時）**
1. 確認攻擊入口並修補漏洞
2. 清除惡意程式或未授權存取
3. 重置受影響帳號與憑證
4. 驗證系統完整性

**Phase 4：復原（24小時-7天）**
1. 從乾淨備份恢復服務（依 RTO 目標）
2. 加強監控（72小時密集觀察）
3. 確認服務恢復正常後，解除事件狀態

### 4.2 設備遺失處置

1. 立即至 IT 服務台回報（分機 200）
2. 資安部門遠端鎖定/抹除設備（MDM）
3. 重置所有在遺失設備上登入過的帳號
4. 評估遺失設備上的資料等級，決定是否需對外通知

## 5. 個資事件通報義務

依據《個人資料保護法》及歐盟 GDPR：

- **台灣個資法：** 發現個資外洩72小時內，須通報主管機關（數位部）
- **GDPR（如涉及歐盟居民資料）：** 72小時內通報監管機關，必要時通知當事人
- 法務部門（legal@twtech.com.tw）必須第一時間介入，確認通報義務

---

## 6. 事後檢討（Post-Incident Review）

所有 P1/P2 事件，須在恢復後5個工作日內完成事後報告（Post-Mortem）。

**報告必須涵蓋：**
1. 事件時間軸（Timeline）
2. 根本原因分析（Root Cause Analysis）
3. 影響評估（受影響系統、資料、客戶）
4. 應對措施評估（哪些做得好，哪些需改進）
5. 改善行動計畫（含負責人與完成期限）

報告提交至 CISO，副本抄送 CTO、COO。
